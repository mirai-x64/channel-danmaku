# チャンネル避け

裏でも番組は流れている。単一 HTML + Canvas、依存なし。

`index.html` をブラウザで開く。マウスで動かす / スペースで回す。

## 賭けの成り立ち

3つの番組は、映っていない間も常に進行している。チャンネルを回すと自機の位置は
そのままに、その番組の進行中の瞬間が出る。だから回した先が空であることも、
いきなり弾の壁であることもある。猶予は構造的に存在しない。

どの番組も自機の位置を参照しない。参照すると回した先が必ず危険になり、
賭けが選択ではなく罰になる。

## 難度をどう決めたか

`tools/sim.mjs` は `index.html` から番組の定義をそのまま抜き出して Node で回す。
自機は反応遅れ 150ms と速度上限を持つ(完璧に避ける AI は死なないので難度を測れない)。

見ているのは一点だけ — **1つの番組に固定したとき、何秒もつか**。
固定して何十秒も生きられる番組があるなら、そこは安全地帯であり、
「回さずに居座る」が最適解になってひねりが死ぬ。

    node tools/sim.mjs 8 60     # 番組ごとの生存秒数
    node tools/sim.mjs margin   # 逃げ場の余裕(px)の分布

この計測で実際に見つかって直したもの:

- **深夜通販** は同心円状に撃っていたため、弾の密度が中心からの距離に反比例し、
  画面の隅が永久に安全だった(固定して 60s 上限に到達)。腕を回す形に変えた。
  半径 d で隙間に留まるのに必要な横速度が ω*d になり、外へ逃げるほど忙しくなる。
- **砂嵐** は最小半径が 46px あり、中心に立っていれば毎周期やり過ごせた。
  加えて隙間が固定だったので、隙間に入って一緒に縮めばよかった。
  最小半径を詰め、隙間を滑らせて動かした(瞬間移動させると壁が自機の上に湧いて
  避けようがなくなるため、必ず滑らせる)。
- **気象通報** は弾が速すぎて画面に残らず、開けた場所で反射だけを試していた。
  遅くして同時に画面上に居る数を増やした。

現状はどの番組も固定で 12〜23s に収まっており、居座れる番組も、
引くと明らかに損な番組も無い。
