# チャンネル避け

裏でも番組は流れている。単一 HTML + Canvas、依存なし。

`index.html` をブラウザで開く。マウスで動かす / スペースで回す。

## 賭けの成り立ち

3つの番組は、映っていない間も常に進行している。チャンネルを回すと自機の位置は
そのままに、その番組の進行中の瞬間が出る。だから回した先が空であることも、
いきなり弾の壁であることもある。猶予は構造的に存在しない。

どの番組も自機の位置を参照しない。参照すると回した先が必ず危険になり、
賭けが選択ではなく罰になる。

## 難度をどう決めたか

長いあいだ「1つの番組に固定して何秒もつか」を AI に測らせていた。これは失敗だった。
その数字は **AI の先読みの上手さ**を測っており、「難しい」と「そもそも避ける手が無い」が
同じ値になる。気象通報が完全知覚・速度2倍でも人間相当と同じ 4.1s だったのを
「盤面が閉じている」と読んだが、実際は閉じていなかった。AI が下手なだけだった。

いま見ているのは、腕前と無関係に盤面だけで決まる3つ。

| 見るもの | 意味 | いまの値 |
|---|---|---|
| 到達可能集合が空になるまで | どんな腕前でも避けられなくなる時刻 | 3番組とも 30s+ で閉じない |
| 動かずに立っていられる場所 | 居座れる安全地帯があるか | 3番組とも 0% |
| 湧いた先の即死率 | 回すことが本当に賭けか | 14% / 16% / 18% |

    node tools/reach.mjs 30 350 one   # 一点から始めて盤面が閉じるか
    node tools/play.mjs smoke         # ブラウザで開いて番組ごとに撮る
    node tools/play.mjs play 8        # 実際のゲームループの上で遊ぶ

`tools/reach.mjs` は前方到達可能集合をそのまま計算する。
R(t+dt) = (R(t) を速度ぶん膨らませたもの) ∩ 空いている場所。
R が空になった瞬間が、どんな腕前でも避けられなくなった瞬間。空にならない限り道は在る。

## この計測で見つかった、番組が番組になっていなかった話

**壁の弾が離れすぎていて、3つとも壁ではなかった。**
自機のドットは半径 3px。隣り合う弾の縁が 6px 以上空いていれば、そこは通れる。
気象通報は間隔 30px / 縁の間 18px で、壁の弾の **100%** がこれを満たしていた。
点線であり、どこでも素通りできた。砂嵐も 93%、深夜通販も 65%。
即死率が上がらないのも、生存が 3s と 31s に割れるのも、これ一つで説明がつく。

弾の間隔は感覚ではなく不等式で決まる —— **間隔 < 2*弾半径 + 2*ドット半径**。

**通路の高さは「休み」の長さで決まる。**
掃射は撃ち終わるまでの間も落ちているので、一枚の壁は高さ `SP*SWEEP` の斜めの坂になる。
壁と壁の縦の間隔は `SP*(SWEEP+REST)` なので、実際に通れる通路は差し引き **`SP*REST`** しかない。
かつて にじみ 147px に対し間隔 168px で、通路は 21px だった。弾の直径 12px を引けば残らない。

**回る点源は外周で必ずほつれる。**
隣り合う2発の角度差 `ω*iv` は撃った瞬間に凍るので、間隔は半径に比例して開く。
深夜通販は隅(d=540)で 52px 開いており、「腕を横切れるのは通り抜け口だけ」が成り立っていなかった。
隅での間隔 `sqrt((sp*iv)^2 + (ω*iv*d)^2)` を 18px 未満に収めるのが条件。

**掃射の進みはフレームではなく1発ごとに進める。**
フレーム単位で進めると、1フレームに2発以上撃つ速さになった途端、
その2発が同じ位置に重なって湧き、壁の間隔が倍に開く。

## 実際に遊んで確かめる

`tools/play.mjs play` は本物のゲームループの上で操作する。当たり判定もクールダウンも
チャンネル切替も本物なので、組み上がった状態で壊れていないかはこれでしか分からない。

8回遊んで 1.8〜11.9s、チャンネルを回したのは 16回、
そのうち死因が「回した直後」だった走行は 3回。回すのは救いでも罰でもなく、賭けになっている。
